DESIGN NOTES · 04 / EVAL RUNS

What's new: A brand-new persisted history view. Every Mode A / Mode B run lands in eval_runs (id, fixture_id, prompt_hash, prompt_snapshot, started_at, finished_at, status, score_breakdown, cost_breakdown, mode, window_size) — this page lists them. Compare two prompts is the killer affordance: tick two checkboxes, hit the rust CTA in the bar, get a side-by-side prompt diff + score delta (in 05-states sheet).

Design choices: (1) Dense monospace table — admin users scan IDs/hashes/times more than they read prose. (2) Status uses the dot + word pattern (no chip) to fit more rows per screen. (3) Filter bar segments mode, status, fixture — three single-axis filters, not a kitchen-sink popover. (4) Each prompt hash is clickable to open the snapshot drawer (state #2 in 05-states). (5) Compare bar swaps copy + becomes rust when armed (2 selected) — strong visual signal.

Reflow < 768 px: Table collapses to cards. Compare bar floats at bottom of viewport when 2 are selected.
