DESIGN NOTES · 03 / NEW FIXTURE

What changed: Textarea is the primary input — no parsed-line list, no per-line collector chip preview. Invalid lines surface as a compact list directly under the textarea. Cancel + Build fixture action bar moves from the top-right page header down to below the textarea + invalid panel, so the user's eye flows top→bottom: name → paste → fix errors → submit.

New submit behavior (grade-first): On a successful build, the fixture file is saved locally (existing) AND the user is redirected to /admin/eval/grade/<new-id> to label each cluster. When Save-to-repo on the grade page completes, the user lands on /admin/eval?fixtureId=<new-id> with the fixture pre-selected and ready for scored evals. Build → grade → eval is the two-hop loop; skipping grading would leave the fixture without ground truth and produce score: null.

Why move the actions: Top-right CTAs work for review pages (where the action applies to the page-wide object) but on a build-step form the action belongs at the bottom of the form it submits — that's where the user's pointer already is after they've finished pasting. The accompanying mono caption ("Next stop: /admin/eval/grade/…") removes ambiguity about what happens next.

Reflow < 768 px: Right rail moves below the form. Action bar stays at the bottom of the left column. Textarea is full-width throughout.
