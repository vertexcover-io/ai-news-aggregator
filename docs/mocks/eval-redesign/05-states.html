DESIGN NOTES · 05 / STATES

Three component-level states used across the eval surfaces: (A) the prompt diff modal at three sizes (empty / small / large with scroll); (B) the run detail drawer that opens from the runs table when you click a run id or prompt hash — left pane is the prompt snapshot rendered as a code block with line numbers, right pane has score + cost breakdowns; (C) the runs-page empty state for first-time users who haven't run an eval yet.

Why this matters: The diff modal already exists but its empty + edge-case states aren't accounted for. The drawer is new — it's the entry point for "view this prompt that was used in run X" and feeds the compare flow on 04. The empty state turns a confusing blank page into a guided next step.
