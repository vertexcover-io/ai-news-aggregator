DESIGN NOTES · 01 / EVAL INDEX

What changed: Prompt editor is now full-width and visually dominant; Mode A controls move to a compact right rail. The "+ New fixture" link is promoted to a clear inline action in the picker row instead of being buried in a label corner. A new aggregate hero sits above the per-fixture results table — four monospace tiles (mean nDCG@10, mean P@10, total cost, run time) — so the headline number is unmissable. Sourcing report uses a stacked horizontal bar per source instead of a plain count table.

Why: The current layout makes the prompt textarea feel cramped on anything below 1280px and forces the user's eye to scan across columns when iterating. Iteration is a textarea-first activity — the editor deserves the prominent slot. Right-rail controls are sticky and don't fight for vertical space with results.

Reflow < 768 px: Editor and controls stack; aggregate hero becomes 2x2; per-fixture table becomes a card list with the metrics tiled inside each card.
