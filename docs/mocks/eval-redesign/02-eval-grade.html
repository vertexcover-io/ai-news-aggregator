DESIGN NOTES · 02 / EVAL GRADE

What changed: Cluster rows are now flat rows inside one bordered table (not card-on-card). Selected state is a 3px rust left-rail instead of a full amber border — quieter, more like a focused row in a code editor. Tier label buttons become numbered 1/2/3 keycap tiles on the right of each row, mirroring the keyboard exactly. The keyboard shortcut bar is pinned under the page head so it's always glanceable. Progress sidebar is a single conic-gradient ring + three tier bars (must / nice / drop) for instant proportion read.

Why: The amber selection in the current build is too loud and fights the labeled-row tints. A rail accent reads as "you are here" without coloring the row. The keycap-tile actions teach the keyboard shortcuts visually — pressing the number on the keyboard mirrors clicking the same-numbered tile. The ring is the single thing the grader needs to glance at: "am I close to done?"

Reflow < 768 px: Right rail collapses into a sticky bottom bar (compact ring + tier counts). Cluster rows lose the thumbnail at <480px; keycap tiles drop below the title.
