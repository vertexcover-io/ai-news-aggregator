How AgentLoop is built

This newsletter writes itself. The harness that wrote it does too.

The Argument

The interesting work in AI right now isn't writing prompts. It's building the harness around the model — the loops, the tools, the evals, the context, the gates.

OpenAI named this harness engineering on their blog earlier this year. The phrase stuck because it pointed at something practitioners already knew: the model is the engine, but the harness is the car. Most of the engineering is in the car.

AgentLoop is our attempt to build a real product — this newsletter — while building the harness it runs on. The two efforts feed each other. Every issue we ship is a stress test for the harness; every bug the harness lets through is a lesson we capture and codify so it can't ship again.

Inspired by OpenAI's harness engineering framing. This is one team's attempt to put it into practice in public.
What it takes to trust AI you don't review

The honest goal of a harness is to ship code humans don't read. That's a strong claim. It only holds if three things hold first.

01

A spec sharp enough to fail against

If "done" is a vibe, the only way to check the work is to read it. We write specs in EARS-format acceptance criteria so the verification step has something concrete to grade. No spec, no shipping.

02

A verification gate that proves it works

Tests and typecheck are necessary, not sufficient. The harness boots the app, runs the feature end-to-end, captures payloads and screenshots, and writes a proof report before any commit. If the gate is weak, the code review you skipped comes back as a bug.

03

Compounding

Every run has to leave the harness sharper than it found it — new tests, new lint rules, new learnings, retired skills. Without compounding, you're not running a harness. You're running a prompt twice.

One feature, one pipeline. Each stage produces an artifact the next stage depends on. The whole thing runs unattended; a human reads the proof report at the end, not the diff.

The compounding loops

The pipeline ships a feature. These loops keep the harness alive.

Weekly

Tech-debt sweep

Every Friday the harness audits the repo for smells it's learned about — N+1 queries, untyped boundaries, missing repository indirection — and opens fix PRs.

Last week: 14 fix PRs · 3 categories · 0 manual triage
Per PR

Doc-sync loop

Before any PR closes, a skill diffs the code against CLAUDE.md and the docs it claims to govern. Stale docs are a known failure mode of AI-authored code; we make the harness pay the cost of keeping them honest.

Coverage: CLAUDE.md · architecture rules · learnings index
Continuous

Skill eval suite

Every skill has a fixture suite it must pass. When a skill misfires in production, the failing case becomes a new fixture and the skill gets updated until it passes. The eval suite is the regression test for the harness itself.

Skills under eval: brainstorm · planning · tdd · code-review · verify · learn
Per repo

Custom linters

Project-specific rules — repository pattern, no relative cross-package imports, collector return shape, dotenv bootstrap order — each one captured the moment a bug taught us we needed it.

Rules in @newsletter/eslint-plugin: 6 · each one is a bug that won't recur
Per bug

Learnings capture

When a review catches something subtle, the lesson becomes a rule file under .claude/rules/learnings/ and ships with every subsequent task. The cache-vs-spec-promise rule, the JS↔SQL alignment rule, the partial-update precondition rule — all came from real bugs and now prevent their own recurrence.

Learnings persisted: 7 and growing
How the newsletter works

The harness is one half. The newsletter pipeline it runs is the other.

Once a day, an agent crawl runs across 34+ sources, dedups, ranks, summarises, and drafts an issue. A human spends a few minutes confirming nothing stupid is about to ship, and then it goes out.

Sources
34+ feeds — HN, Reddit, Twitter, RSS, GitHub, company blogs. See the live reading list →
Collect
Each source has a collector. The harness wrote most of them.
Enrich + dedup
Links are fetched, summarised, and collapsed to one row per story.
Shortlist
A cheap LLM pass picks ~30 candidates from a few hundred.
Rank + recap
A second pass scores the shortlist on novelty, signal-vs-hype, actionability, and writes the recap you read.
Review
A human spends ~4 minutes per issue confirming the agent didn't ship something stupid.
Ship
Archive page, email, LinkedIn, X — all from one run.
