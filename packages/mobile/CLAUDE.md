# @newsletter/mobile

Expo (managed) React Native app — the "AgentLoop Dispatch" phone app. The mobile analogue of
`@newsletter/extension`: a logged-in operator shares ANY link (via the OS share sheet) or pastes
a URL, and it becomes a `manual`-sourced candidate in the next newsletter run.

## Shape
- `app.json` — Expo config. `scheme: agentloopdispatch` (required for share intent). The
  `expo-share-intent` config plugin sets up the Android intent filter (`text/*`) + iOS share
  extension; `expo-secure-store` plugin for the Keychain/Keystore token. `newArchEnabled`.
- `src/App.tsx` — root. Wraps `ShareIntentProvider`; a tiny state machine routes
  loading → `LoginScreen` | `AddScreen` | `ConfirmationScreen`. A shared URL pre-fills AddScreen
  (re-keyed on the URL so a fresh share remounts the form).
- `src/screens/` — `LoginScreen` (password → bearer token), `AddScreen` (URL + optional title →
  "Add to next issue"), `ConfirmationScreen` (success / already-queued).
- `src/api.ts` — `POST /api/extension/login` + `/submissions` with `Authorization: Bearer`.
  Mirrors the extension client. `API_BASE` from `EXPO_PUBLIC_API_BASE` (`src/config.ts`).
- `src/storage.ts` — token in `expo-secure-store` (NOT AsyncStorage). On a 401, clear + return to login.

## Rules
- Reuses the SAME backend as the extension: the `ext|`-namespaced HMAC bearer path
  (`/api/extension/*`), NOT the admin cookie. No backend change is needed — native fetch isn't
  subject to CORS, so the `chrome-extension://`-scoped CORS gate doesn't apply.
- Token lives in `expo-secure-store`, never plaintext AsyncStorage / `localStorage`.
- Own toolchain: this package is excluded from the repo-root ESLint flat config (strict
  type-checked, Node-oriented). Lint/typecheck via the package scripts (eslint-config-expo +
  expo/tsconfig.base). Don't try to fold it into the root TS project graph.
- Share intent + secure store need a **dev build** (`expo prebuild` + `expo run:*` or an EAS
  dev-client build) — they don't work in Expo Go. `pnpm start:go` is paste-only smoke testing.

## Commands
pnpm --filter @newsletter/mobile start        # expo start --dev-client (needs a dev build)
pnpm --filter @newsletter/mobile start:go     # expo start (Expo Go; no share intent / secure store)
pnpm --filter @newsletter/mobile ios          # expo run:ios  (local prebuild + build)
pnpm --filter @newsletter/mobile android      # expo run:android
pnpm --filter @newsletter/mobile typecheck    # tsc --noEmit
pnpm --filter @newsletter/mobile lint         # eslint .

## Monorepo / pnpm
`metro.config.js` is tuned for pnpm: `watchFolders` = repo root, `nodeModulesPaths` =
[package, root], `disableHierarchicalLookup`. If Metro can't resolve a module after install,
add `node-linker=hoisted` to a root `.npmrc` and reinstall. `ios/` + `android/` are generated by
prebuild and gitignored (managed workflow).
