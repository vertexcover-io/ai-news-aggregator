# Tech-debt ignore rules — code-duplication in test files

## Scope

This file suppresses `code-duplication` findings where EVERY instance of the clone
group lives inside a test file (regex: `\.test\.|\.e2e\.|/tests?/|__tests__|\.spec\.`).

Test scaffolding repetition (fixture builders, mock objects, assertion helpers) is
expected and intentional — normalising it into shared helpers is typically high
noise/low value and the duplication is isolated to the test layer.

**Coverage:** 737 findings across 239 test files derived from
`.harness/tech-debt/2026-06-04/findings.json` via jq/Python using the test-path
regex above.  Only findings where the finding's own `path` AND every path extracted
from the `detail` field match the test-path regex are suppressed.  Any finding where
even one instance is in production code is NOT suppressed here.
