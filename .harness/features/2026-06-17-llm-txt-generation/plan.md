# Plan: llm.txt / llms.txt Generation

TDD throughout. Phases below; the phase graph is linear (each builds on the prior).

```dot
digraph phases {
  rankdir=LR
  "P1: shared generator" -> "P2: API endpoints" -> "P3: materialization script" -> "P4: snapshot + docs"
}
```

## Phase 1 — Shared pure generator (`@newsletter/shared/llm-txt`)

**Files:**
- `packages/shared/src/llm-txt/render.ts` — pure functions.
- `packages/shared/src/llm-txt/types.ts` — `IssueMeta`, `LlmTxtSite`, `LlmTxtStaticPage` input types.
- `packages/shared/src/llm-txt/index.ts` — barrel.
- `packages/shared/src/llm-txt/__tests__/render.test.ts` — VS-1..VS-4 + empty states.
- Add subpath export `@newsletter/shared/llm-txt` to `packages/shared/package.json` exports + tsup
  entry (follow existing `scheduling` subpath pattern).

**Functions (data-in / string-out, no I/O):**
- `absoluteUrl(baseUrl, pathOrUrl)` — join, dedupe slashes, pass through already-absolute URLs.
- `renderIssueLlmTxt(issue: IssueMeta, stories: LlmTxtStory[], opts: { baseUrl }): string`
  - `LlmTxtStory` is the subset of `RankedItem` we need: `{ title, url, recap }` (recap optional).
- `renderCanonLlmTxt(entries: PublicMustReadEntry[], opts: { baseUrl }): string`
- `renderIndexLlmsTxt(input: { site, issues: IssueIndexEntry[], canon, staticPages, opts }): string`
- `renderIndexLlmsFullTxt(input: { ...same..., issuesFull: { meta, stories }[] }): string`

**Static page content** lives as a small constant array in `llm-txt/static-pages.ts`
(`{ title, path, description }`): "How we build it" (`/`), "Archive" (`/archive`), "Sources"
(`/sources`), "Canon / Must-read" (`/must-read`). Site title + summary constants too.

Tests cover VS-1..VS-4 and VS-6 empty path. Pure → fast unit tests, no DB.

## Phase 2 — API endpoints (`@newsletter/api`)

**Files:**
- `packages/api/src/routes/llm-txt.ts` — `createLlmTxtRouter(deps)` + `createDefaultLlmTxtRouter()`.
  - `deps`: `getArchiveRepo`, `getRawItemsRepo`, `getMustReadRepo`, `baseUrl`, `logger`.
  - Routes (router mounted at root for index files, and per-issue handled via archives path):
    - `GET /llms.txt` — fetch recent reviewed archives (repo `listReviewed`), canon
      (`mustRead.listPublic`), build via `renderIndexLlmsTxt`. `c.body(text, 200, headers)`.
    - `GET /llms-full.txt` — same + hydrate each issue's ranked items
      (`hydrateRankedItems`) → `renderIndexLlmsFullTxt`.
    - `GET /archives/:runId/llm.txt` — fetch archive, 404 if not reviewed; hydrate ranked items;
      `renderIssueLlmTxt`. (Mounted under `/api/archives` so URL is `/api/archives/:id/llm.txt`.)
- Wire into `packages/api/src/app.ts`:
  - Add `llmTxtIndexRouter` (root: `/llms.txt`, `/llms-full.txt`) mounted via `app.route("/", ...)`.
  - Add the per-issue route to the public archives router OR a small `llmTxtArchiveRouter` mounted
    at `/api/archives` BEFORE the `:runId` catch-all (mirror how `/search` is mounted first).
  - Extend `BuildAppDeps`.
- Wire into `packages/api/src/index.ts`: construct with `newsletterBaseUrl` and default repos.
- Tests: `packages/api/src/routes/__tests__/llm-txt.test.ts` — VS-5, VS-6, VS-7 using in-memory
  Hono `app.request(...)` with mock repos (mirror `archives-route.test.ts`).

**Mounting note:** root paths `/llms.txt` and `/llms-full.txt` are mounted directly on `app` in
`buildApp` before the admin gate, exactly like `/health`. Per-issue uses the existing `/api/archives`
prefix.

## Phase 3 — Materialization script

**Files:**
- `packages/api/scripts/generate-llm-txt.ts` — connects via default repos, fetches issues + canon,
  hydrates, calls the shared generator, writes the `llms/` tree at repo root using `fs`.
  - Resolves base URL via `resolveBaseUrls(process.env)`.
  - Writes: `llms/llms.txt`, `llms/llms-full.txt`, `llms/canon.llm.txt`,
    `llms/issues/<issueDate>-<runId>.llm.txt`.
  - Idempotent; clears/overwrites the `issues/` dir contents it manages.
- Root + api `package.json`: add `"generate:llm-txt"` script (`tsx scripts/generate-llm-txt.ts`).
- Test: `packages/api/src/routes/__tests__/llm-txt-drift.test.ts` — VS-8: build index from the same
  mock inputs the route uses and assert the string equals the route body (no-drift). Pure-ish; no FS.

## Phase 4 — Initial snapshot + docs

- Generate an initial `llms/` snapshot. In dev without DB data, write a committed **example**
  snapshot from representative fixture data so the tracked files exist and are reviewable. (If a
  local DB with seed data is available, generate from it.)
- `.gitignore`: ensure `llms/` is NOT ignored (it's a deliverable). Add a short `llms/README.md`
  explaining the files are generated by `pnpm generate:llm-txt` and how prod serves the live
  versions.
- Update `packages/api/CLAUDE.md` route table note + the root README/docs if a route list exists.

## Risks / mitigations

- **SPA can't serve root `/llms.txt`** → served by API; prod rewrite documented (out of code).
- **Drift between endpoint and file** → single shared generator + VS-8 no-drift test.
- **Repo-access lint rule** → generator takes data, not DB; endpoints/script use repo factories.
- **Hydration cost on full index** → cap issues inlined in `llms-full.txt` to a recent N (e.g. 30).
